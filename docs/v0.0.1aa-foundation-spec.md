# v0.0.1aa Foundation Specification

## Scope statement

Version `v0.0.1aa` establishes the first end-to-end, local-first baseline for Atlas Library.

### Goal

A user can build a small world map with linked articles and reopen it reliably.

## Phase 1 baseline deliverables included in v0.0.1aa

1. Project create/open/save
2. Infinite pan/zoom canvas
3. Image import/place/move/scale/rotate/opacity
4. Text blocks with installed OS fonts
5. Basic layers panel
6. Fullscreen toggle
7. Minimal codex viewer/editor (markdown raw + rendered)
8. Internal/external anchors (basic creation + click navigation)

## Technical design baseline

- Camera transform: world-space ↔ screen-space
- Immediate culling: render only visible nodes
- Basic selection model: single + multi
- Initial SQLite schema
- `.atlaslib` project bundle read/write

## Recommended architecture layout

```text
com.atlaslibrary
 ├─ app/
 ├─ ui/
 ├─ domain/
 ├─ editor/
 ├─ render/
 ├─ codex/
 ├─ linking/
 ├─ indexing/
 ├─ storage/
 └─ integration/
```

Design rule: `domain` avoids UI-specific dependencies.

## v0.0.1aa acceptance criteria

- A sample project can be created and saved as `.atlaslib`.
- The saved project can be reopened with map nodes and linked codex articles intact.
- Internal anchor navigation routes to target article or map object by stable ID.
- External anchor navigation requires explicit user action.

## Minimal persistence model (initial schema target)

- `projects`
- `boards`
- `layers`
- `nodes`
- `node_props_image`
- `node_props_text`
- `articles`
- `anchors`
- `settings`

## Version track policy for this project

- Every release marker must include a two-letter suffix.
- Suffix is required (not optional) and acts as a unique identifier per push stream.
- Auto-advance rule:
  - Increase version track every 5 to 10 pushes (configurable by workflow policy).
  - Keep suffix progression continuous and deterministic.

## Next checkpoints after v0.0.1aa

- `v0.0.1ab` to `v0.0.1az`: implementation hardening and missing deliverable completion.
- Move to `v0.1.0aa` when Phase 1 exit criteria are met and verified.
